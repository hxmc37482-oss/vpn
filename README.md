# MartVPN Bot

Телеграм-бот с оплатой через CryptoBot, выдает VLESS/подписку-ключ после оплаты.

## Установка

```bash
pip install -r requirements.txt --break-system-packages
```

## Баннер

Положи картинку приветствия в файл:

```
banner/welcome.jpg
```

Если файла нет — бот просто отправит текст без картинки, ничего не сломается.

## Запуск

```bash
python bot.py
```

## Важно перед запуском на Replit

- Токены уже вписаны в `config.py` (бот-токен и CryptoBot токен) — раз ты их скинул в чат, где угодно кроме своего приватного репозитория/деплоя их лучше не светить, и по-хорошему потом токен бота можно перевыпустить через @BotFather (/revoke), а токен CryptoBot — через настройки приложения в @CryptoBot, если переживаешь что кто-то мог их увидеть.
- `payments.json` создается автоматически, в нем хранится история инвойсов — на Replit не забудь чтобы файл не удалялся при рестарте (или замени на постоянную БД, если будет много юзеров).

## Как это работает

1. `/start` → баннер + кнопка "Купить доступ"
2. Юзер жмет "Купить" → бот создает счет в CryptoBot → присылает кнопку "Оплатить" (ссылка) и "Я оплатил, проверить"
3. Юзер оплачивает, жмет "Я оплатил, проверить" → бот дергает CryptoBot API, если оплачено — присылает VLESS-ключ

## Важное предостережение

Ключ, который выдает бот, **статичный и общий на всех покупателей** (это одна и та же подписка на VPN-сервис, а не индивидуально создаваемый доступ). Это значит:

- Все оплатившие юзеры фактически шарят один и тот же канал — при большом числе клиентов скорость и стабильность будут падать
- Если сервис-провайдер (Ben's VPN) не разрешает перепродажу доступа в своем ToS — есть риск, что аккаунт забанят, и тогда все клиенты разом останутся без доступа

Для нормального масштабируемого бизнеса стоит перейти на свой VPS с 3x-ui/Xray, где бот через API создает **отдельный** конфиг под каждого клиента.
